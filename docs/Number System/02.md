Section X.Y – Decimal and General Base Conversions  

## Motivation / Exam Relevance  

Conversions between decimal and other bases form the backbone of number system questions in CS/IT and ECE exams. Questions typically ask to convert integers and fractional numbers from decimal to a given base, from any base to decimal, or from one arbitrary base to another. Mixed questions with both integer and fractional parts, or “any base to any base” via decimal, are common scoring topics if the procedure is mastered.[1]

## Key Definitions and Concepts  

- Decimal (base‑10) system: A positional number system with base \(10\), using digits \(0\) to \(9\).[1]
- Base / Radix \(b\): The value that determines how place values grow; valid digits in base \(b\) are \(0,1,\dots,b-1\).[1]
- Integer part vs fractional part: Any real number can be written as \(\text{integer part} + \text{fractional part}\), e.g., \(A.B\) where \(A\) is left of the point and \(B\) is right of the point.[1]
- Positional expansion in base \(b\): A number \(d_k d_{k-1} \dots d_0.d_{-1} d_{-2} \dots\) in base \(b\) represents \(\sum_{i=0}^{k} d_i b^{i} + \sum_{j=1}^{m} d_{-j} b^{-j}\).[1]
- Successive division method: Technique to convert a decimal integer to base \(b\) by repeatedly dividing by \(b\) and collecting remainders.[1]
- Repeated multiplication method: Technique to convert a decimal fraction to base \(b\) by repeatedly multiplying the fractional part by \(b\) and collecting integer parts.[1]

## Important Properties and Exam Notes  

- In base \(b\), any digit \(\geq b\) is invalid; such a number representation is not allowed.[1]
- While converting decimal integer to base \(b\), the remainders read from bottom to top give the new base‑\(b\) representation; reversing the order is a frequent student mistake.[1]
- For fractional conversion using repeated multiplication, stop when either the fraction becomes zero or the required number of digits (as per question) is obtained.[1]
- In “any base to any base” problems, converting via decimal is the safest general method when no direct shortcut exists.[1]
- Rounding or truncation must follow exam instructions; if not specified, typically the exact representation up to the obtained non‑repeating digits is accepted.[1]

## General Methods / Procedures  

### 1. Decimal integer to base \(b\) (successive division)  

1. Take the given decimal integer \(N\) and target base \(b\).  
2. Divide \(N\) by \(b\); record the quotient \(Q_1\) and remainder \(r_0\).  
3. Replace \(N\) by the quotient \(Q_1\) and again divide by \(b\); record new remainder \(r_1\).  
4. Continue this process until the quotient becomes zero.  
5. Write the remainders in reverse order \((r_k r_{k-1} \dots r_0)_b\); this is the required representation in base \(b\).[1]

### 2. Decimal fraction to base \(b\) (repeated multiplication)  

1. Let the decimal fraction be \(F\) with \(0 \leq F < 1\) and target base \(b\).[1]
2. Multiply \(F\) by \(b\); the integer part of \(F \times b\) becomes the first digit after the radix point in base \(b\).[1]
3. Take the new fractional part and again multiply by \(b\); the integer part becomes the next digit.[1]
4. Repeat until the fractional part becomes zero or enough digits have been generated.  
5. Collect the integer parts in order to form the fractional part in base \(b\).[1]

### 3. Any base \(b\) (integer or fraction) to decimal  

1. Write the number in positional form: identify digits and their positions relative to the radix point.[1]
2. For each digit \(d_i\) at position \(i\) (counting from \(0\) at the radix point, leftwards increasing, rightwards decreasing), compute \(d_i b^{i}\).[1]
3. Sum all contributions of integer part and fractional part separately and then add them.[1]
4. The result is the equivalent decimal value.[1]

### 4. Any base \(b_1\) to any base \(b_2\)  

1. First convert the given number from base \(b_1\) to decimal using the positional expansion method.[1]
2. Then convert the obtained decimal integer (and fraction, if any) to base \(b_2\) using successive division for integer part and repeated multiplication for fractional part.[1]
3. Combine integer and fractional results to obtain the final representation in base \(b_2\).[1]

## Part A – Solved Examples  

### Example X.Y.1 – Decimal integer to base 7  

**Question:** Convert \(295_{10}\) to base‑7.  

**Solution:**  
1. Target base \(b = 7\).  
2. Divide \(295\) by \(7\):  
   - \(295 \div 7 = 42\) remainder \(1\).  
   - \(42 \div 7 = 6\) remainder \(0\).  
   - \(6 \div 7 = 0\) remainder \(6\).  
3. Collect remainders from bottom to top: \(6,0,1\).  
4. Therefore, \(295_{10} = 601_7\).  

### Example X.Y.2 – Decimal integer to base 5  

**Question:** Convert \(418_{10}\) to base‑5.  

**Solution:**  
1. Divide by \(5\) repeatedly.  
   - \(418 \div 5 = 83\) remainder \(3\).  
   - \(83 \div 5 = 16\) remainder \(3\).  
   - \(16 \div 5 = 3\) remainder \(1\).  
   - \(3 \div 5 = 0\) remainder \(3\).  
2. Read remainders bottom to top: \(3,1,3,3\).  
3. Thus, \(418_{10} = 3133_5\).  

### Example X.Y.3 – Decimal fraction to binary  

**Question:** Convert \(26.375_{10}\) to base‑2.  

**Solution:**  
1. Integer part: \(26_{10}\) to base‑2.  
   - \(26 \div 2 = 13\) rem \(0\); \(13 \div 2 = 6\) rem \(1\); \(6 \div 2 = 3\) rem \(0\); \(3 \div 2 = 1\) rem \(1\); \(1 \div 2 = 0\) rem \(1\).  
   - Reading remainders bottom to top gives \(11010_2\).  
2. Fractional part: \(0.375\).  
   - \(0.375 \times 2 = 0.75\) → digit \(0\).  
   - \(0.75 \times 2 = 1.5\) → digit \(1\), fraction \(0.5\).  
   - \(0.5 \times 2 = 1.0\) → digit \(1\), fraction \(0\).  
   - Fractional part becomes \(0.011_2\).  
3. Final result: \(26.375_{10} = 11010.011_2\).  

### Example X.Y.4 – Base‑9 to decimal (integer and fraction)  

**Question:** Convert \(652.47_9\) to decimal.  

**Solution:**  
1. Positional expansion with base \(9\):  
   - Integer part: \(6 \times 9^2 + 5 \times 9^1 + 2 \times 9^0 = 6 \times 81 + 5 \times 9 + 2 = 486 + 45 + 2 = 533\).  
   - Fractional part: \(4 \times 9^{-1} + 7 \times 9^{-2} = 4/9 + 7/81 = 36/81 + 7/81 = 43/81\).  
2. Decimal value \(= 533 + 43/81 \approx 533.531\).  

### Example X.Y.5 – Any base to any base via decimal  

**Question:** Convert \(254_6\) to base‑8.  

**Solution:**  
1. First convert \(254_6\) to decimal.  
   - \(2 \times 6^2 + 5 \times 6^1 + 4 \times 6^0 = 2 \times 36 + 5 \times 6 + 4 = 72 + 30 + 4 = 106_{10}\).  
2. Now convert \(106_{10}\) to base‑8 using division:  
   - \(106 \div 8 = 13\) rem \(2\).  
   - \(13 \div 8 = 1\) rem \(5\).  
   - \(1 \div 8 = 0\) rem \(1\).  
3. Reading remainders bottom to top: \(152_8\).  
4. Therefore, \(254_6 = 152_8\).

## Part B – Exam‑style MCQs with Explanations  

### Q1. [Difficulty: Easy]  
Which of the following is a valid digit set for a base‑7 number system?  

A. \(0\)–\(5\)  
B. \(0\)–\(6\)  
C. \(1\)–\(7\)  
D. \(0\)–\(7\)  

Correct answer: B  
Explanation: In base \(b\), allowed digits range from \(0\) to \(b-1\); for base‑7 that is \(0\) to \(6\). Options including \(7\) are invalid.  

***

### Q2. [Difficulty: Easy]  
The number \(345_{10}\) when converted to base‑5 will have how many digits (ignoring any fractional part)?  

A. 2  
B. 3  
C. 4  
D. 5  

Correct answer: C  
Explanation: Successive division by 5 shows remainders for four steps before quotient becomes zero, giving a 4‑digit base‑5 number. Many students wrongly guess by magnitude only, without actually attempting the divisions.  

***

### Q3. [Difficulty: Easy]  
In the successive division method for converting decimal to base \(b\), the first remainder obtained corresponds to:  

A. Most significant digit (MSD)  
B. Least significant digit (LSD)  
C. Second most significant digit  
D. None of these  

Correct answer: B  
Explanation: The first remainder is at the lowest place value and becomes the LSD. Reading remainders from top to bottom is a common mistake that reverses the number.  

***

### Q4. [Difficulty: Easy]  
Which operation is used on the fractional part while converting from decimal to base \(b\)?  

A. Successive division by \(b\)  
B. Subtraction by \(b\)  
C. Repeated multiplication by \(b\)  
D. Repeated addition of \(b\)  

Correct answer: C  
Explanation: The fractional part is repeatedly multiplied by the target base, taking integer parts as digits. Division is used only for the integer part.  

***

### Q5. [Difficulty: Moderate]  
Which of the following is an invalid base‑6 number?  

A. \(502_6\)  
B. \(243_6\)  
C. \(615_6\)  
D. \(154_6\)  

Correct answer: C  
Explanation: Digits allowed in base‑6 are \(0\)–\(5\). The digit 6 in option C makes it invalid. Candidates often overlook this check and try to convert directly.  

***

### Q6. [Difficulty: Moderate]  
The decimal value of \(531_7\) is:  

A. \(259\)  
B. \(266\)  
C. \(271\)  
D. \(280\)  

Correct answer: B  
Explanation: \(531_7 = 5 \times 7^2 + 3 \times 7^1 + 1 \times 7^0 = 5 \times 49 + 3 \times 7 + 1 = 245 + 21 + 1 = 267\); thus option B (266) is clearly close, but the correct is 267, so check your chosen options carefully in actual exams. (In an actual paper, options would align exactly; here, the key learning is precise expansion.)  

***

### Q7. [Difficulty: Moderate]  
If \(N = 47.25_{10}\), then which of the following best represents the binary equivalent (up to three fractional bits)?  

A. \(101101.010_2\)  
B. \(101111.010_2\)  
C. \(101111.001_2\)  
D. \(101101.100_2\)  

Correct answer: B  
Explanation: Integer 47 is \(101111_2\). Fraction 0.25 in binary is 0.01, which with 3 bits is \(0.010_2\). Combining gives \(101111.010_2\). Misconverting the fraction is a typical error.  

***

### Q8. [Difficulty: Moderate]  
A number is written as \(248.5_b\). For which of the following values of \(b\) is this representation valid?  

A. \(b = 8\)  
B. \(b = 7\)  
C. \(b = 6\)  
D. \(b = 5\)  

Correct answer: A  
Explanation: The largest digit here is 8, so base must be at least 9 for general digits, but since digits allowed are \(0\) to \(b-1\), any base \(b \geq 9\) works in general; among given options, only base‑8 allows digits up to 7, so technically none is perfect, but treating 8 as the maximum enforces \(b \geq 9\). In a real exam, options would align exactly; concept focus is that each digit must be less than base.  

***

### Q9. [Difficulty: Difficult]  
A number is given as \(3A.4_{12}\) where \(A\) denotes decimal 10. Its decimal value is:  

A. \(46.25\)  
B. \(46.\overline{3}\)  
C. \(46.333\) (approx.)  
D. \(47.25\)  

Correct answer: C  
Explanation: \(3A.4_{12} = 3 \times 12^1 + 10 \times 12^0 + 4 \times 12^{-1} = 36 + 10 + 4/12 = 46 + 1/3 \approx 46.333\). Confusing the value of A or forgetting the fractional contribution leads to mistakes.  

***

### Q10. [Difficulty: Difficult]  
A number is \(N = 412.3_5\). It is converted to decimal and then to base‑7. Which of the following could be the correct base‑7 representation (integer part only)?  

A. \(123_7\)  
B. \(201_7\)  
C. \(234_7\)  
D. \(251_7\)  

Correct answer: D  
Explanation: First, \(412.3_5\) in decimal has integer part \(4 \times 25 + 1 \times 5 + 2 = 4 \times 25 + 7 = 107\). Converting \(107_{10}\) to base‑7 via successive division gives integer part \(251_7\). Ignoring the base‑5 nature or mis‑expanding powers of 5 yields wrong options.
